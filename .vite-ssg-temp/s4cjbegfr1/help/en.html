CSL Style Generator Help

Hi,
using the app may seem a bit daunting at first but don’t worry: there are just a few easy to understand concepts you’ll learn in no time!

So here’s what you need to know:

  • Angle brackets (<> ) are used for variable placeholders. <title> for example will display the title.
    You can find a list of all available variable placeholders down below.

  • Square brackets ([]) are used for names.
    The syntax for names is a bit more involved but quite intuitive still:
    You first specify the type of name you want to display and then the way you display it.
    Right now there are three types of name available: author, editor and contributor.
    Let’s say John Doe authored a book. You can display his name in three different ways:

    1. only the family name, e.g. Doe
      code: [author<family>]
    2. first given and then family name, e.g. John Doe
      code: [author<given><family>]
    3. family name, given name, e.g. Doe, John
      code: [author<family><given>]

    Abbreviating the given name is handled through a separate checkbox in the Settings section.

  • Curly brackets ({}) are used for optional parts of your styling.
    Let’s say you want to cite a chapter of a book. Your code includes a placeholder for the edition of the book like so:
    [author<family>], <title>, in: [editor<family>] (ed.), <volumeTitle>, <place>: <publisher>, <edition>, <year>, S. <pages>
    But not all chapters in your database have an edition attached. If you happened to cite one of these, there would remain a “hull” of sorts around your <edition> placeholder, in this case a comma and a whitespace.
    To avoid this we can wrap our <edition> placeholder with everything that is only supposed to be displayed if this is present in our database in curly brackets like so: {, <edition>}. Now the comma and whitespace will only be displayed if there is an edition attached to the chapter.

Important:

Optional parts are shown on the condition that the first specified variable exists.
For example: {, <volumeTitle>, <edition>} will be shown if <volumeTitle> exists but regardless of wether <edition> exists or not.

  • Parentheses (()) are used to specify the font style that is used. There are three options available:

    • (i) will make the text italic.
    • (b) will make the text bold.
    • (sc) will put the text in small caps.

    Combining the modifiers is also possible. They are used inside the angle brackets of a placeholder. Two examples:

    • <title(b)> will display the title in bold.
    • [author<family(i)(b)(sc)>] will display the authors family name in bold, italic and small caps.
